# Junior Fullstack-аналитик — решение задания

## Цель
Подготовить аналитические артефакты для разработки функционала: фронтенд + бэкенд (учебный кейс).

## Что сделано
- Сформулированы требования (Functional / Non-functional)
- Описаны пользовательские сценарии (Use Cases / User Stories)
- Зафиксированы бизнес-правила и ограничения
- Описаны интеграции и контракты API (черновая спецификация)
- Описаны сущности и модель данных (ER / таблицы / поля)
- Добавлены сценарии проверок / acceptance criteria (как критерии готовности)
